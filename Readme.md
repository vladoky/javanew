##Пример запуска кластера Ignite в Docker Swarm окружении

Запускается 1 опорный узел (может быть несколько), остальные узлы - масштабируемые серверные подключаются к опорному через доменное имя (tasks.ignite-master в ignite-config.xml)
В контейнеры через swarm передается конфиг игнайта, в клиентский ignite-контейнер со spring boot приложением передается доменное имя опорного узла через переменную окружения IP_FINDER_ADDRS

В случае Docker Swarm деплоймента (командой stack) пробрасываемые порты 
    
    ports:
      - "8080:8080"
      - "5005:5005"
публикуются для адреса 0.0.0.0 (в случае виртуальной машины - ip адрес виртуальной машыны в сети виртуальных машин). 
Для удобства отладки? чтобы можно было обращаться к сервису как localhost:8080 настройки в stack.yml определены следующим образом:
    
    ports:
      - mode: host
        target: 8080
        published: 8080
      - mode: swarm
        target: 5005
        published: 5005
И поэтому порт 8080 будет доступен по интерфейсу loopback

Запуск:

    docker stack deploy --compose-file stack.yml test
    docker service ls

В логах могут мелькать SYSTEM_WORKER_BLOCKED (на master) и прочие Failed to wait for initial partition map exchange, но через несколько минут после запуска все должно устаканиться, а test_app ксервис должен напечатать сводку о стабильном состоянии топологии 3-й версии:
Если этого не произошло и ошибки продолжают печататься -удалить (docker stack rm  test) и передеплоить пока в логах app не появится Completed partition exchange. По-хорошему нужно будет, если будет время, отладить последовательность первоначального запуска, чтобы друг за другом с интервалами запускались master, slaves и app
